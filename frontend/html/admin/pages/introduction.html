Introduction

Welcome to MDC Builder Administration

There is an overwhelmingly large amount of data within a mission, and this administration interface hopefully makes life easier and to help ensure data is kept up to date in a reasonable way

Structure

Fundamentally there is a straight forward hierarchy in place to avoid having to re-input data over and over again

  • Theatres
    • The base element to build upon
    • extracted from DCS via LUA, most people won't need to touch this too much
    • As DCS doesn't have pragmatic way to obtain ICAO, AI ATC Frequencies or TCN, these are manually set here
    • You are also able to edit display names of airfields (e.g: Abu Dhabi Intl => Abu Dhabi)

  • Squadrons
    • A Squadron is a container of other information
    • Theatre Overrides
      • Mainly ATC frequencies for Base DCS airfields to be inherited by missions
      • PAR, ATS, GND, TWR, CTRL, TCN etc.
    • Members + Borts
    • Flight Callsigns
    • Frequency Codes (for lookups)
    • Rather manual!

  • Missions
    • A mission is part of a squadron and is the front-end selector
    • Inherits infromation from squadron and theatre
    • Updates using the DCS miz file directly from your browser
    • Allows the following overrides
      • Bullseye extracted from Miz (As no name, you can set here)
      • DCS Airfield Frequencies from Squadron
      • Mission Airfields (FARPs, Carriers etc)
      • Navpoints
        • We extract navpoints from several sources within a miz
        • nav_points: these are coalition specific
        • triggers: some missions (BlueFlag for instance) use a lot of triggers which help identify locations etc. so they're useful to extract
        • custom: manual entries
        • You can hide entries you don't want users selecting
      • Aircraft Presets
        • We extract presets for each coalition from the miz
        • Read Only as you should update the miz
        • Warnings
          • If the miz has conflicting presets on aircraft, a warning section will be shown with the AC that need fixing
          • The "correct" value is determined by the frequency list with the most AC
          • I.e: if 5 AC have presets [1,2,3] and 2 AC have [1,2,4] and 1 AC has [1,2,5] then [1,2,3] will be used and the 3 incrrect AC will be identified
